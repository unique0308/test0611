## 🔴 Tier 1 · 往死里测(坏了 = 用户丢活儿 / 平台没法用)

### 1.1 资产硬删 —— 最危险,不可恢复
路径:左侧栏「资产」(`/assets`)
- [ ] 删 1 个任务 → 确认产物图真没了,且 `/profile` 看板"本月已用"数字跟着减对
- [ ] 框选多个瓦片 → 批量删 → 弹的提示是不是"**全部产物一并移除,不可恢复**"
- [ ] 删一个**被收藏过**的任务 → "我的收藏"里那张也消失(解绑),没残留破图/死链
- [ ] 切到**别的员工身份** → 确认看不到、也删不到不属于自己的任务
- 🔎 *重点:有没有"删了 A 结果 B 也跟着没了"的级联误伤;删完刷新页面状态是否一致*

### 1.2 主标签必选 blocking —— 刚上(M5 波 2),最容易拦错
路径:生成页(`/`),先点「创建新对话」开一个**全新会话**
- [ ] 新会话第一次:**没选主标签**就点生成 → 应该**被拦住、生成不了**(头部提示先选主标签)
- [ ] 在**会话头部 chip(📌)**选一个主标签:营销推广 / 产品展示 / 设计参考 / 汇报演示 / 其他 → 现在能生成
- [ ] 生成后,Dock 里「使用目的」默认 = 你刚选的那个主标签
- [ ] 在 Dock 把「使用目的」**单次改成**别的 → 胶囊变 **warn 色 + 前面带 ⚡**(悬停显示"本次覆盖")→ 这次用新的,但**会话主标签没被改**(再生成一次,默认还是原来的)
- [ ] 回头部 chip 改主标签 → 只影响**之后**的新生成默认值,**不动已经生成的历史**标签
- [ ] 主标签选「其他」→ 弹一个 **<20 字短文本**框 → 填超过 20 字被截断;**留空也能提交**

### 1.3 生成主链路 × 4 场景
路径:生成页(`/`)
- [ ] **文生图**:写 prompt → 选模型 / 比例 / 张数 → 生成 → 看状态(排队中 → 已完成)→ 出图 → 下载
- [ ] **多图**:张数选 2/3/4 → 出对应张数 → 每张能单独下载 / 放大
- [ ] **图生图**:上传参考图(仅 PNG/JPEG/WebP,≤20MB)→ 生成
- [ ] **文生视频**:切到视频 → 选时长(5s / 10s)→ 生成 → 出的视频能播
- [ ] **错误上传**:传 >20MB 或非图片 → 友好报错,不崩
- [ ] **取消**:生成中点「停止生成」→ 状态变"已取消"(注意它会提示:请求发出后上游已开始计费)
- 🔜 *本节后续会随"生成面板布局优化 + prompt 增强"扩(R2)*

## 🟠 Tier 2 · 会话 + 收藏(新改的,最容易状态错位)

### 2.1 会话化跨态
- [ ] 生成几条后**刷新**页面 → feed 还在(不是刷新就丢)
- [ ] 同一会话开**两个标签页** → 一边生成,另一边刷新能看到
- [ ] 「创建新对话 / +新对话」→ 跳到空会话(hero 态)
- [ ] 会话**重命名** → 试 emoji / 超长 / 带换行 → 名字不崩、截断正常(约 18 字)
- [ ] **「默认创作」**会话:确认**不能删、不能改名、不能置顶**
- [ ] 置顶一个会话 → 跑到列表最上;置顶多个时按时间排
- [ ] 删一个会话 → 里面任务的处理符合预期,**收藏不应跟着一起死**

### 2.2 收藏 output 级(单张粒度)
- [ ] 一个 **4 图任务**,点**第 3 张**的 ⭐ → 只有第 3 张被收藏(不是整任务、不是默认收第 1 张)
- [ ] 资产页勾「我的收藏」筛选 → 只显示已收藏的那一张
- [ ] 取消收藏 → 状态正确
- [ ] 列表视图的 ⭐(对每行首张 output 0)跟画廊视图的收藏状态一致

## 🟡 Tier 3 · 看板数字(给老板看的地基,错了最伤)
路径:`/profile`(切员工)、`/admin`(切 admin)、`/manager`(切 manager)
- [ ] 自己生成 N 次后,`/profile`「本月已用 / 约等于多少次」对得上实际
- [ ] `/admin` KPI「本月调用 / 活跃部门 / 成本」跟你实际生成量一致
- [ ] 配额进度条的 % = 已用 / 额度,百分比算得对
- [ ] `/manager` 只看到**本部门**数据(换成别的部门 manager,看不到跨部门)
- [ ] 报销数据统计的金额加总 = 各笔之和
- 🔎 *重点:任何"看板数字跟你手数对不上"的,一律记 bug —— 这是要进老板汇报材料的,不能有错*

## 🟢 Tier 4 · 报销 + 异常输入

### 4.1 报销
- [ ] 提交申请:工具名称 / 金额 / 使用周期 / 使用说明 / 上传凭证 → 单笔 **>¥2000 被拦**
- [ ] admin 审核:通过 / 驳回(驳回要填原因)→ 员工侧状态跟着变(审核中 → 通过 / 驳回)
- [ ] 一次传**多张凭证** / 传非图片凭证 → 处理正常不崩

### 4.2 异常输入(低成本高产出)
- [ ] 空 prompt / 纯 emoji / 超长 prompt → 不崩、有合理反馈
- [ ] **狂点生成按钮** → 应只允许 1 个任务在跑(不能并发出一堆)
- [ ] 各种下拉 / 弹窗:快速开关、按 Esc 关、点外部关 → 都正常
