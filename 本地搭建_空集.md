# 本地搭建手册(空集)

> 目标:在你**自己的电脑**上把 AI 中台跑起来,用来做自测(见 `自测清单.md`)。
> 这套是**纯 mock 配置 + 你自己的数据库**,跟嘉斌的线上/数据完全隔离 —— 你随便点、随便删,不会影响别人。
> ⚠️ 你拿到的是 mock 环境:登录是假的(点选身份)、出的图是占位图,**这是正常的**,本轮不看出图质量。

---

## 0. 你需要先有

- **Node.js 18+**(建议 20)。命令行敲 `node -v` 能看到版本就行。
- **Git**。`git --version` 能看到就行。
- 一个 **Supabase 免费账号**(下面第 3 步现注册,5 分钟,免费)。

---

## 1. 拿代码

嘉斌会发你一个项目压缩包(`ai-platform-给空集.zip`)。下载后**解压**,在命令行进入这个文件夹:

```bash
cd ai-platform
```

## 2. 装依赖

```bash
npm install
```

## 3. 建你自己的数据库(关键 · 隔离用)

1. 打开 https://supabase.com → 注册 → **New Project**(免费 Free 档)。
2. 项目建好后,去 **Project Settings**,抄下面 4 个值(下一步要填):
   - `Settings → API`:**Project URL**、**anon/public key**、**service_role key**
   - `Settings → Database → Connection string`:**URI**(形如 `postgresql://postgres:...@...supabase.co:5432/postgres`)

> 为什么要你自己的库:这样你删数据、我改表互不干扰。**别用嘉斌的库。**

## 4. 配置 `.env.local`

在 `ai-platform` 根目录新建文件 `.env.local`,把下面整段贴进去,**只改 4 个 Supabase 值 + 1 个 SESSION_SECRET**,其余保持原样:

```bash
NEXT_PUBLIC_APP_URL=http://localhost:3000

# 随机 32 字节:命令行跑 `openssl rand -base64 32` 把输出贴这
SESSION_SECRET=粘贴一段随机串

# ↓↓↓ 填你自己 Supabase 的 4 个值(第 3 步抄的)
NEXT_PUBLIC_SUPABASE_URL=你的_Project_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的_anon_key
SUPABASE_SERVICE_ROLE_KEY=你的_service_role_key
DATABASE_URL=你的_数据库_URI

# ↓↓↓ 全 mock,保持原样别改
AUTH_MODE=mock
STORAGE_MODE=local
NOTIFICATION_MODE=mock
MONITORING_MODE=noop
EASYROUTER_MODE=mock

# admin 白名单:让 jiabin 这个 seed 账号有管理员权限(测看板用)
ADMIN_EMAILS=jiabin@example.com
```

> 其余的飞书 / 火山 / OSS / Sentry key **全部不用填** —— mock 模式不读它们。

## 5. 建表(把 schema 灌进你的库)

```bash
npm run db:migrate
```

这会把 24 个 migration 跑进你的 Supabase(**幂等**,重复跑没事)。跑完你的库里就有了表 + seed 数据(6 个测试员工等)。

## 6. 起服务

```bash
npm run dev
```

浏览器开 **http://localhost:3000**。

## 7. 登录 + 切身份

mock 登录:开 **http://localhost:3000/auth/dev**,点选一个身份进去。seed 的 6 个测试账号:

| 邮箱 | 角色 |
|---|---|
| `jiabin@example.com` | **管理员**(测 /admin 看板、报销审核用这个) |
| `zhangsan@example.com` / `lisi@example.com` / `wangwu@example.com` / `zhaoliu@example.com` / `qianqi@example.com` | 普通员工 |

> Tier 1 资产硬删那条"切别人身份验权限",就是用这里换账号验证。

## 8. 开测

跟着根目录 **`自测清单.md`** 一条一条走,发现问题按里面的 bug 模板记。

## 常见问题

| 现象 | 处理 |
|---|---|
| `npm run db:migrate` 报连不上 | 检查 `.env.local` 的 `DATABASE_URL` 抄对没(尤其密码段) |
| 3000 端口被占 | `npm run dev -- -p 3001`,然后开 localhost:3001 |
| 出的图是灰块/占位 | 正常,mock 不出真图,本轮不看图质量 |
| 页面要登录 | 先开 `/auth/dev` 选身份 |
| 拿不到代码 | 找嘉斌把压缩包发你 |
