## 4. AI 洞察路线图（下一阶段规划）

> **2026-05-26 用户拍板**：所有 AI 洞察走「**独立 AI 总结页**」，不在仪表盘卡片内嵌建议。
> 设计理由：① 决策心智分离（仪表盘看状况 / AI 页看下一步做什么）；② AI 内容需要更多空间展开主张+证据+影响估算；③ 可统一管理状态（忽略/已执行/待跟进/重新触发）。

### AI 总结页结构（待落地）

```
/admin/insights （独立 tab 或 sidebar 入口）
├ Top 紧急（4-5 条，红色优先）
├ 分类区
│   ├ 配额策略（超额预警 / 冗余推荐 / 调整建议）
│   ├ 模型策略（异动 / 续订 / 新模型采购）
│   ├ 支出策略（预算预测 / 工具优化）
│   └ 用户异常（突增 / 异常模式）
└ 每条洞察展开：主张 / 证据（含跳回看板链接）/ 影响估算 / 操作（一键跳处理 / 忽略 / 标已处理）
```

**重要约束**：仪表盘里**不嵌建议**（保持仪表盘"看状况"的纯粹性），洞察统一进 AI 页。

### 三层洞察按"实现成本 vs 决策价值"分层

第 1 层规则引擎可立刻落地，第 2/3 层等数据积累或人工拍板。

### 第 1 层 · 规则引擎（推荐先做）

无需 LLM，纯规则 + SQL，可解释性强、零幻觉。

| 洞察 | 用什么数据 | 输出形式 |
|---|---|---|
| **配额超额预测** | quotas + 当月消耗 + 剩余天数线性外推 | "X 部门按当前速度，本月 28 日将超额" |
| **异常单点突增** | task records + 滑动窗口 z-score | "李梦琪 本周积分是上周的 4.2 倍" |
| **模型偏好漂移** | model_top 月度对比（**当前已有 modelTopMom 数据**） | "Flux Pro 月环比 +156%，建议续订前评估" |
| **配额冗余/紧缺** | quotas + usage history | "市场部 月使用率 26%，建议下调配额" |

**落地位置**：admin overview 第一屏底部 alert 区（之前删过的"需要关注"位置可以重生为"规则洞察"）。

> **2026-05-28 M3 决策更新（D14）**：上表"阈值告警版"是 V2 目标态。
> V1 阶段改走"**信号呈现版**"（不拍阈值，让 admin 看排序自己判断）：
> - I-1 部门用量速度雷达 → 不说"将超额"，说"预计月末用 N 积分"
> - I-2 员工变化 Top 10 → 不说"突增"，说"本周是上周 X 倍"
> - I-3 模型异动 → 复用现有"模型异动"模块连接到 Insights
> - I-4 部门配额健康度 → 进度条渐变（绿/黄/红 by 已用%，不是阈值）
>
> **落地位置升级**：从"admin overview 底部 alert 区" → **独立 `/admin/insights` 页面**
> （决策心智分离，看板看状况 / Insights 看下一步）
>
> V1 → V2 演化路径：跑 1-2 月看真实分布定阈值 → 启用预留的"阈值上线"控件 → 升级为上表的"阈值告警版"
>
> 详见 `MVP跟踪文档/产品总图.md § 8 D14`

### 第 2 层 · LLM 总结（中等成本）

需要 LLM API（GPT-4 / Claude），用结构化数据 + 严格 prompt 减少幻觉。

| 洞察 | 用什么数据 | 输出形式 |
|---|---|---|
| 月度报告自动生成 | 整月汇总数据 → LLM | 自然语言摘要 |
| 部门看板智能旁注 | 单部门聚合数据 → LLM | "该部门 90% 用 Flux Pro 做海报，建议..." |
| 用途标签智能合并 | purpose tags + 文本相似度 | "海报 / 海报设计 共 220 次，建议合并" |

### 第 3 层 · 趋势预测 + 主动策略（最贵，最有价值）

需要历史数据 ≥3-6 个月 + 时间序列预测（ARIMA / Prophet / LLM）。

| 洞察 | 输出 |
|---|---|
| 下月预算预测 | "预计下月 ¥31k（同比 +12%），建议预算 ¥35k" |
| 采购包优化 | "AI 检测到 70% 用户只用其中 3 个工具 → 砍 5 个，年省 ¥168k" |
| 配额政策推演 | What-if 模拟：「如设计组配额 60k→80k，预计月支出 +¥1200，但减少 1 起停滞」 |

## 6. 设计原则备忘（避免回退）

1. **决策导向 > 信息堆砌**：每个模块必须能驱动某个 admin 决策；纯展示数据但无行动价值的应该砍掉或合并
2. **派生值不显示**：avg、per-capita 等可数学推算的值不占独立卡，由用户心算或显示在 hint
3. **告警嵌入主指标**：超额 / 临近上限 不放独立 alert 区，**塞进相关 KPI 卡的 foot 文本**，看一眼主页就知道哪要紧
4. **数据 vs 装饰**：信息密度型（表格 / KPI）保持紧凑；装饰展示型（目的分布 / 模型偏好等列表）用大标题 + 宽行距 + 粗进度条
5. **模型异动 > 模型排名**：单纯 Top N 信息密度低；环比 delta 才有决策价值
6. **同名指标的层级区分**：上下两层都用 KPI 卡时，下层必须**形态截然不同**（如卡 → inline stat 横条）+ **label 加层级前缀**（如部门名），消除字面重复造成的认知混淆
7. **AI 洞察走独立总结页**：仪表盘保持"看状况"纯粹性，不在卡片内嵌建议；所有 AI 建议统一进 `/admin/insights`（待建）
8. **重要工具条保持可见**：长列表场景的筛选 / 批量操作 / 分页都用 sticky 钉住屏幕边界，admin 滚动浏览不丢失上下文
9. **可点击反馈用"色泽晕影 + 上提"**：避免 hover 时变换背景色或加 inset ring（视觉太躁），统一用「`transform: translateY(-1~2px)` + `box-shadow` 软扩散光晕」表达可点击；普通用 `--accent-glow`，告警态用 warn glow；hover 时若元素有边线，可让 `borderRadius` 微微变大让浮起的元素脱离容器线框
